# デザイン方針（Design）

## 目的

本アプリの視覚・操作に関する方針を定義し、UI 実装およびスタイル決定時の判断基準を提供する。

## 主な内容

- デザインガイドライン（アクセシビリティ、視覚階層、アニメーションルール）
- カラーパレット / タイポグラフィ
- コンポーネントの視覚ルール（カード、表、ボタンの振る舞い）
- レスポンシブ対応方針

---

## デザインガイドライン

- **装飾制限**  
  過剰な装飾は採用しない。目的が明確な場合のみ限定的に使用する。
- **要素の目的重視**  
  表示要素は必ず指標や操作に結びつくこと。
- **最小限の視覚階層**  
  色とタイポグラフィは情報優先度を示すために使用する。
- **アクセシビリティ優先**  
  キーボード操作、十分なコントラスト、aria 属性を考慮する。
- **アニメーションの用途限定**  
  状態変化やユーザーフィードバックに限定して使用する。

---

## レスポンシブ対応方針

- 本アプリは **レスポンシブ対応を行わない**。
- 想定利用環境は **デスクトップ（PC）** のみとする。
- モバイル・タブレット向けのレイアウト最適化やブレークポイント設計は行わない。
- 画面サイズ差異への対応は、最低限の崩れ防止（overflow 制御など）に留める。

---

## タイポグラフィ

- **フォント**  
  Google Fonts を使用する。
- **ベースフォントサイズ**  
  16px を基準とする。
- **方針**  
  フォントサイズや太さは、可読性と情報階層を明確にする目的でのみ変更する。

---

## カラー・テーマ（ライト / ブラック）

### 目的

利用者が視認性や好みに応じてライトモードとブラックモードを切り替えられるようにし、アクセシビリティと一貫した視覚階層を維持する。

### 要件

- ユーザー操作でライト／ブラックモードを切り替えられること（UI にトグルを用意する）。
- 切替状態は永続化しない（localStorage を使用しない）。
- CSS カスタムプロパティ（変数）を用いて色設計を行い、テーマ切替は変数の切り替えで実現する。
- コントラスト比は WCAG AA 基準を満たす（主要テキストおよびインタラクティブ要素）。

### 実装方針

- テーマ管理はグローバル state（`theme: 'light' | 'dark' | 'system'`）で管理する。デフォルトは `system`。
- 永続化は行わない。起動時は `prefers-color-scheme` を参照して初期化する。
- CSS 層では `:root` にライト用カラートークンを定義し、ダーク時は `data-theme="dark"` を `html` または `body` に付与して上書きする。

### カラートークン例

- `--bg`（背景）
- `--surface`（カード / 表の背景）
- `--text`（本文）
- `--muted`（補助テキスト）
- `--primary`（アクセント / ボタン）
- `--danger`
- `--success`

---

## UI / 配置

- テーマ切替トグルはアプリのヘッダ右上に配置する。
- トグルはアイコンとラベルを併記し、キーボード操作可能にする（フォーカス指標を明確に）。

---

## アクセシビリティ

- テーマ間でコントラスト比を検証する（主要テキストは 4.5:1 以上を目標）。
- 色だけに依存しない状態表現（アイコンやラベル）を維持する。
